5.5.3、配置参数怎么获取
1、通过ApplicationConfig
类：com.edk4j.core.config.ApplicationConfig
将本类用@Autowired引入，并调用其get方法，可获取的参数见edk4j.properties文件中"application."前缀的配置
2、通过CacheConfig
类：com.edk4j.core.config.CacheConfig
将本类用@Autowired引入，并调用其get方法，可获取的参数见edk4j.properties文件中" cache."前缀的配置
3、通过DataBaseProperties
类：com.edk4j.core.config.DataBaseProperties
将本类用@Autowired引入，并调用其getDataBaseName()方法，可获取application.properties配置文件中的spring.jpa.database字段的值（数据库名称：ORACLE,MYSQL,DB2）
4、通过FileUploadConfig
类：com.edk4j.core.config.FileUploadConfig
将本类用@Autowired引入，并调用其get方法，可获取的参数见business.properties文件中" multiPartResolver."前缀的配置
5、通过RedisCacheConfig
类：com.edk4j.core.config.RedisCacheConfig
将本类用@Autowired引入，并调用其get方法，可获取的参数见application-{profile}.properties文件中" redis."前缀的配置
6、通过SessionConfig
类：com.edk4j.core.config.SessionConfig
将本类用@Autowired引入，并调用其get方法，可获取的参数见edk4j.properties文件中" sessionConfig."前缀的配置
7、通过ProxyConfig
类：com.edk4j.core.integration.config.ProxyConfig
将本类用@Autowired引入，并调用其get方法，可获取的参数见application-{profile}.properties文件中" proxy."前缀的配置
8、实现接口HttpIntegrationConfig
接口：com.edk4j.core.integration.config.HttpIntegrationConfig
实现该接口，定义http接口配置，具体配置类需要在具体应用中实现
可获取的参数见application-{profile}.properties文件中"自定义具体接口的配置前缀."前缀的配置
edk-integration中对接口参数的调用都是通过接口的get方法获取的
9、实现接口SocketIntegrationConfig
接口：com.edk4j.core.integration.config.SocketIntegrationConfig
实现该接口，定义socket接口配置，具体配置类需要在具体应用中实现
可获取的参数见application-{profile}.properties文件中"自定义具体接口的配置前缀."前缀的配置
edk-integration中对接口参数的调用都是通过接口的get方法获取的
10、具体项目中还有扩展内容，需要怎么配置

在business.properties中新增了下面内容
bankSettings.bankName=XXX银行
bankSettings.bankTelephoneNo=12345678

spring在加载时会自动将properties中的配置值注入到对象中

@Component

@ConfigurationProperties(prefix = "bankSettings", ignoreInvalidFields = true)

public class BankSettingsConfig {

      /**银行名称**/

      private boolean bankName;

      /**银行电话号码**/

      private boolean bankTelephoneNo
