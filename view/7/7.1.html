7.1、RequestPresenter接口
1、需要实现的接口
：com.edk4j.core.requestprocess.RequestPresenter
：详见前面“edk中的api.xml”小节内容
，由于每个api请求都可以配置RequestPresenter，因此你可以根据不同的需要，编写不同的RequestPresenter来处理。
：com.edk4j.sdk.web.presenter.DefaultRequestPresenter，该类在edk4j-demos项目中可以找到。也可以使用EDK4j可视化IDE为项目添加支持后，自动生成。
▶  必须实现的方法清单： 
/**
 * 调用doHandle服务方法前的处理逻辑，完成安全检查、权限检查等一系列动作，返回sessionContext
 * @param HttpServletRequest servletRequest
 * @param context 前台请求数据都已经设置到context中了
 * @param api的配置实体类
 * @return 返回sessionContext
 */
public abstract Context preHandle(HttpServletRequest servletRequest, Context context, Api api) throws BaseException;
/**
 * 调用doHandle方法后的处理逻辑，完成sessionContext的保存处理等动作
 * @param response 
 * @param api的配置实体类
 * @param context 注意：这里是交易context，不是sessionContext
 * @return resultString json或者xml格式的字符串
 * @throws BaseException
 */
public abstract String postHandle(HttpServletResponse response, Api api, Context context) throws BaseException;

7.1.2、如何组织返回结果数据

在<businessLogic>中配置好<output>之后，在postHandle方法中处理返回结果：

（1）返回json时这样写

String result = super.getResponse(BaseConstant.JSON_ FORMAT, api, context);

return result;

（2）返回xml时这样写

String result = super.getResponse(BaseConstant.XML_ FORMAT, api, context);

return result;

（3）返回其他格式的结果时，需要自行组织处理！
